Regression¶

For regression the general prediction fromula look like:

$$ \hat{y}=w[0]*x[0]+w[1]*x[1]+...+w[p]*x[p]+b $$

the x's are the fetures of a single data point, w and b are paramters of the model that are learned. $\hat{y}$ is the prediction of the model. For a dataset with a single feature this becomes

$$ \hat{y}=w[0]*x[0]+b $$

To start our investigation we will look at learning paramters from the wave dataset

Ridge regression¶

The additional constraint of ridge regression is that the magnitudes of the weights should be as small as possible while still predicting the data well. This constraing is and example of regularization. Regularization means explicitly restricting a model to avoid overfittiing. Ridge regression uses L2 regularization. Lets see how ridge regression preforms on the boston housing data set:

In [9]:
from sklearn.linear_model import Ridge
ridge = Ridge().fit(X_train, y_train)
print("Training set score: {:.2f}".format(ridge.score(X_train, y_train))) 
print("Test set score: {:.2f}".format(ridge.score(X_test, y_test)))

Training set score: 0.89
Test set score: 0.75

The ridge model makes a trad-off between simplicity and preformance on the training set. this trade off can be changed by using the alpha parameter. Increasing alpha forces coeficients to move toward zero which decreases training set preformance but might help generalization, for example:

By letting alpha be very small we end up not restricting the weights much at all and end up with a model that resembles linear regression:

Lasso Regression¶

An alternative to ridge regression is lasso regression. Again lasso also restricts the weights close to zero but does it through L1 regularization. With this L1 regularization some of the weights are exactly zero. This means some of the features are ignored and can be seen as a form of automatic feature selection. This can also helo to ilicit the most important features of the model. Lets apply this method to the boston housing data:

In [15]:
from sklearn.linear_model import Lasso
lasso = Lasso().fit(X_train, y_train)
print("Training set score: {:.2f}".format(lasso.score(X_train, y_train))) 
print("Test set score: {:.2f}".format(lasso.score(X_test, y_test))) 
print("Number of features used:", np.sum(lasso.coef_ != 0))

Training set score: 0.29
Test set score: 0.21
Number of features used: 4

Lasso does quite poorly on the training and test sets, this indicates we are underfitting. The model is only using 4 of the 104 features. Lasso also has a regularization parameter alpha. By default this parameter is 1 . To reduce underfittin we can try and decrease alpha but we also need to increse the default setting of max_iter

In [16]:
# we increase the default setting of "max_iter",

lasso001 = Lasso(alpha=0.01, max_iter=100000).fit(X_train, y_train) 
print("Training set score: {:.2f}".format(lasso001.score(X_train, y_train))) 
print("Test set score: {:.2f}".format(lasso001.score(X_test, y_test))) 
print("Number of features used:", np.sum(lasso001.coef_ != 0))

Training set score: 0.90
Test set score: 0.77
Number of features used: 33

The lower alpha allows us to fir a more complex model. We have slightly better preformance than the ridge model and are only using 33 fetures. This may make the model potentially easier to understand. If alpha is to low we remove the effect of regularization and end up over fitting. In this case we get a similar resuklt to linear regression:

When alpha=1 we see that most of the coefficients are zero as we already knew but also the remaining coeeficients are also small. Whne. alpha =.01 we see many features are still zero but the non-zero weights still have a large magnitude. When alpha=.001 we have a highly unregularized model where not many of the weights are 0. the ridge madel with alpha=.1 is simal in predictive power as the lasso model with alpha=.01 but the ridge model still has all the weights as non-zero.

In practice ridge regression is usually the first choice between the two. If you know that your data has many features but most of them will not be important lasso might be a better choice. Lasso also provides a model that is easier to understan as it will only select a subset of the input features. there is also a class call ElasticNet in scikit-learn, which combines the L1 and L2 penalties. This works best but there are now multiple paramters to adjust to fine tune your model.
