Decision trees¶

Decision trees are widely used models for classification and regression tasks. Essen tially, they learn a hierarchy of if/else questions, leading to a decision. You can think of a deciosion tree as a game of asking wuestions to distinquish between different classes. Imagine you want to distinguish between the following 4 animals: bears, hawks,penquins, and dolphins. you strat with an if/else question this splits the animals into two groups then from there you ask a different question on the remainig groups to seperate into 4 greoups:

This process creates a binary tree of decisions and each node contains a test. Another way of thinking about it is that each test partitions the feature space along one of the axes. The algorithm can be seen as building a hierarchial partition. Each test concerns a single feature and the resulting partitons have axis parallel boundaries. This processs can be repeated until each partition (also called leafs) onle contains a single value. These leaves are called pure leaves. To use the algorithm on a new data point we follow the tree until we reach the leaf where the new data point would be. The output would be the majority of points in that leaf.

This method can also be used for regression. To make predictions we traverse the tree to the leaft that would contain the point. The output of the data point would be the average of the terget data coantianed in that leaf.

Controlling complexity of decision trees¶

If you run the algorithm until all leaves are pure then you get a modle that is very complex and highly overfitt to the training data. The presence of pure leaves means that the model is 100% accurate on the training data. This can be seen in the figure above where on the left there are regions defined as being class 1 among all the class 0 points. Also on the right there is a single stip defined as class 0 among the class 1 region. This is not how we would imagine the decision boundary to look, the boundaries are focused on single outlier points that are far away from other points.

There are two common stratagies to prevent overfitting: stopping the tree early ( aaslo called pre-pruning) or building the tree and then collapsing nodes that contain little information (also called post-pruning or just pruning). Possible criteria for pre-pruning include limiting the max-depth of the tree, limiting the maximum number of leaves, or requiring a ninimum number of points in a node to keep splitting it.In scikit-leran only pre-pruning is implemented. We can look at the effect of pre-pruning using the breast cancer dataset:

Regression Models¶

We have focused on decision trees for classification but have noted that they could be used for regression. Trees for regression are implemented in DecisionTreeRegressor in scikit-learn. The important difference when using trees for regression is that they can not extrapolate, or make decisions outisode the range of the training data. To look at this we will consider historical prices of RAM data:

Strengths, Weaknesses, and Parameters¶

The paramter that controls the model complexity in trees is pre-pruning of the tree. This is done by picking either max_depth, max_leaf_nodes, or min_samples_leaf. Decision trees have two main adavatages over others discussed so far. The results are easy to visulizes and explain and they scale invariantly to the size of the problem. Decision trees work weel wehn the features have many different scales or there is a mis of binary and continous features.

The main doenfall of decision trees is that even with pre-pruning they tend to over fit and provide poor generalization. In practice what works well is creating an ensemble of decision trees.
